Last-Try OCR — System Architecture

End-to-end flow: PDF input → intelligent routing → OCR → validation → structured JSON output

① Master Pipeline Flow

flowchart TD
    A([PDF Input]) --> B[pipeline.py]
    B --> C[Generate Doc ID]
    C --> D[Create Output Dirs]
    D --> E{Multiprocessing?}
    E -- Yes --> F[ProcessPoolExecutor]
    E -- No --> G[Sequential]
    F --> H[process single page]
    G --> H
    H --> I[pdf router - Detect page type]
    I --> J{Page Type?}
    J -- digital --> K[Extract digital text - PyMuPDF]
    J -- scanned --> L[Render page to PNG at 200 DPI]
    K --> M[unicode validator - Validate text]
    M --> N{Validation Result?}
    N -- PASS --> O[Use digital text - Engine PyMuPDF]
    N -- CORRUPTED --> P[Render to image - Route to API chain]
    N -- FAIL --> L
    P --> Q[Step 1 Gemini - Direct OCR]
    Q --> R{Gemini Success?}
    R -- Yes --> S[Engine = Gemini]
    R -- No --> QQ[Step 2 Ollama - Direct OCR final fallback]
    QQ --> RR{Ollama Success?}
    RR -- Yes --> SS[Engine = Ollama]
    RR -- No --> L
    L --> T[ocr engine - Local OCR - PaddleOCR or EasyOCR]
    T --> U[bn and en recognition]
    U --> V[confidence scorer - Multi-signal scoring]
    V --> W{Confidence Score?}
    W -- HIGH --> X[Keep local OCR result]
    W -- LOW --> Y[Step 1 Gemini - Fallback OCR]
    Y --> Z{Gemini Success?}
    Z -- Yes --> AA[Engine = Gemini]
    Z -- No --> YY[Step 2 Ollama - Final Fallback OCR]
    YY --> ZZ{Ollama Success?}
    ZZ -- Yes --> AAA[Engine = Ollama]
    ZZ -- No --> X
    O --> AB[Post-Processing]
    S --> AB
    SS --> AB
    X --> AB
    AA --> AB
    AAA --> AB
    AB --> AC[bangla corrector - Stages A to D]
    AC --> AD[numeric validator - Fix digit confusion]
    AD --> AE[table handler - Detect and parse tables]
    AE --> AF[image processor - Extract embedded images]
    AF --> AG[PageResult assembled]
    AG --> AH[json builder - Save per-page JSON]
    AH --> AI[All pages done]
    AI --> AJ[json builder - Merge to document JSON]
    AJ --> AK([Final Output - Merged JSON])

② Unicode Validation Detail

flowchart LR
    A([Raw Digital Text]) --> B[Count chars]
    B --> C{Enough text - 30 plus chars?}
    C -- No --> SCAN([Treat as Scanned])
    C -- Yes --> D[control char ratio]
    D --> E{More than 5 percent?}
    E -- Yes --> CORRUPT([Corrupted Font - Skip EasyOCR - Gemini Direct])
    E -- No --> F[cid reference count]
    F --> G{CID refs present?}
    G -- Yes --> CORRUPT
    G -- No --> H[winansa artifact count]
    H --> I{Legacy font artefacts?}
    I -- Yes --> WARN([Warning - Fall to OCR])
    I -- No --> J[bracket density - Bijoy font check]
    J --> K{Bracket density above 5 percent?}
    K -- Yes --> WARN
    K -- No --> L[symbol noise ratio - Suspicious glyphs]
    L --> M{Noise above 12 percent?}
    M -- Yes --> WARN
    M -- No --> PASS([Valid Text - Use Digital])

③ Confidence Scoring (Multi-Signal)

flowchart TD
    A([Local OCR Blocks])
    A --> B[Signal 1 - Avg OCR Confidence - weight 0.30]
    A --> C[Signal 2 - Bangla Unicode Ratio - weight 0.15]
    A --> D[Signal 3 - Dictionary Match Rate - weight 0.20]
    A --> E[Signal 4 - Suspicious Glyph Ratio - weight 0.15]
    A --> F[Signal 5 - Numeric Consistency - weight 0.10]
    A --> G[Signal 6 - Structural Consistency - weight 0.10]
    B --> H[Weighted Sum 0.0 to 1.0]
    C --> H
    D --> H
    E --> H
    F --> H
    G --> H
    H --> I{Is Bangla-heavy?}
    I -- Yes --> J{Score above 0.65?}
    I -- No --> K{Score above 0.55?}
    J -- Yes --> KEEP([Keep Local OCR])
    J -- No --> FALL([API Fallback - Gemini then Ollama])
    K -- Yes --> KEEP
    K -- No --> FALL
    style KEEP fill:#14532d,color:#fff,stroke:#22c55e
    style FALL fill:#7c3aed,color:#fff,stroke:#a855f7

④ Bangla Correction Pipeline (4 Stages)

flowchart LR
    IN([Raw OCR Text]) --> A
    A[Stage A - Unicode Normalization - NFC and invisible char removal] --> B
    B[Stage B - Dictionary Validation - Word validity score] --> C
    C{Validity below 60 percent?}
    C -- No --> D
    C -- Yes --> D
    D[Stage C - Pattern Correction - matra and hasanta fixes - digit confusion fixes] --> E
    E{Is Bangla-heavy AND low validity?}
    E -- No --> OUT
    E -- Yes --> F
    F[Stage D - Gemini LLM Validation - Send text to Gemini for Bangla proofreading] --> G
    G{Gemini Response OK?}
    G -- Yes --> H[Replace text with Gemini-corrected version]
    G -- No --> OUT
    H --> OUT([Corrected Text])
    style F fill:#7c3aed,color:#fff,stroke:#a855f7
    style OUT fill:#14532d,color:#fff,stroke:#22c55e

⑤ API Fallback Chain Detail - Gemini then Ollama

flowchart TD
    A([Low-Confidence or Corrupted Page Image]) --> B[Encode PNG as Base64]
    B --> C[Build structured OCR prompt with Bengali numeral guide]
    C --> D[Step 1 - Call Gemini API - google-generativeai]
    D --> E{Gemini response ok?}
    E -- No retry --> D
    E -- Fail or quota or no key --> OL1[Step 2 - Check Ollama - ollama serve running?]
    E -- Yes --> F[Parse JSON from Gemini]
    F --> G{Valid blocks?}
    G -- No --> OL1
    G -- Yes --> J([Engine = Gemini])
    OL1 --> OL2{Ollama running?}
    OL2 -- No --> FAIL([All engines failed - keep local OCR result])
    OL2 -- Yes --> OL3[Detect vision model - llava or minicpm-v or moondream]
    OL3 --> OL4{Vision model found?}
    OL4 -- No --> FAIL
    OL4 -- Yes --> OL5[Send image and prompt to Ollama api/generate]
    OL5 --> OL6{Ollama response ok?}
    OL6 -- Timeout or error --> FAIL
    OL6 -- Yes --> OL7([Engine = Ollama - Final fallback succeeded])
    style D fill:#7c3aed,color:#fff,stroke:#a855f7
    style OL1 fill:#c2410c,color:#fff,stroke:#f97316
    style OL5 fill:#c2410c,color:#fff,stroke:#f97316
    style J fill:#14532d,color:#fff,stroke:#22c55e
    style OL7 fill:#14532d,color:#fff,stroke:#22c55e
    style FAIL fill:#7f1d1d,color:#fff,stroke:#ef4444

⑧ Simple Processing Outline - How It Works

flowchart TD
    IN([File Input - PDF or Image]) --> LANG{Language Detection}
    LANG -- English or Mixed --> ENG[Digital text extraction - PyMuPDF direct]
    LANG -- Bangla scanned --> SCAN[Render to image for OCR]
    ENG --> VALID{Text valid?}
    VALID -- Yes --> DONE([Use digital text])
    VALID -- No Corrupted font --> STEP1
    SCAN --> STEP1
    STEP1[Step 1 - Local OCR - PaddleOCR then EasyOCR] --> C1{Confidence OK?}
    C1 -- Yes High confidence --> OUT([Use local OCR result])
    C1 -- No Low confidence or zero blocks --> STEP2
    STEP2[Step 2 - Gemini API - Cloud vision model] --> C2{Gemini Success?}
    C2 -- Yes --> OUT2([Use Gemini result])
    C2 -- No API key missing or quota exceeded --> STEP3
    STEP3[Step 3 - Ollama FINAL FALLBACK - Local vision LLM - llava or minicpm-v] --> C3{Ollama Success?}
    C3 -- Yes --> OUT3([Use Ollama result])
    C3 -- No Ollama not running or no vision model --> FALLOUT([Keep best local OCR result or empty page])
    OUT --> POST
    OUT2 --> POST
    OUT3 --> POST
    FALLOUT --> POST
    POST[Post-processing - Bangla correction - Numeric fix - Table detection - JSON build]
    POST --> FINAL([Structured JSON Output])

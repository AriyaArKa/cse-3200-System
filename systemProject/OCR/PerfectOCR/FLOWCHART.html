📦 Module Architecture

⚡ TurboOCREngine
turbo_pipeline.py
Core v3.0 engine. Native PDF upload to Gemini Files API. Single API call for entire document. Parallel fallback with ThreadPoolExecutor(5). JPEG 85% @ 200 DPI for image fallback.
📦 BatchProcessor
turbo_pipeline.py
Multi-PDF batch processing. Shared TurboOCREngine instance. Progress callback per file. Streaming generator mode for real-time results.
🖥️ Turbo Dashboard
turbo_app.py
Clean Streamlit UI. No user options — auto-optimized. Multi-file upload. Progress tracking, download buttons (JSON/Text). Per-file expandable results with tables.
⚙️ Config
config.py
Central configuration. API keys (GEMINI, OPENAI from .env). Models: gemini-2.5-flash, gpt-4o. DPI=250, strategy=gemini_primary. Correction=gemini-2.5-flash. Paths, timeouts, cache settings.
🧠 OCR Engines
ocr_engines.py
GeminiOCREngine & GPT4oOCREngine. MASTER_PROMPT: Bengali numeral protocol (6-step), handwriting rules, image descriptions. OCRUsageTracker for call counting. Used by legacy pipeline.
🏃 Fast OCR
fast_ocr.py
Tesseract-based local OCR (FREE). English-only bypass — skip API calls. should_use_ai_ocr() decision function. Confidence threshold: 75%. Tesseract path: D:\3-2\Ollama\tesseract.exe
🔗 Merger
merger.py
Merges Gemini + GPT-4o results. Gemini primary — GPT-4o substitutes low-confidence blocks. Table merge: keeps richer version. Form + notes merging. Used in "dual" strategy.
🔧 Correction
correction.py
Bangla text post-correction. Fixes mattra, hasanta, conjunct errors. Bengali numeral validation (date checks). Uses gemini-2.5-flash. Corrects full text + individual low-confidence blocks.
🔄 Pipeline
pipeline.py
Legacy orchestrator — PerfectOCRPipeline. 5 strategies: dual, gemini_primary, gpt4o_primary, *_only. Hybrid: Tesseract → Gemini → GPT-4o fallback. Full correction + save pipeline.
📄 PDF Processor
pdf_processor.py
PyMuPDF (fitz) based. PDF → PNG images with configurable DPI. PageImage dataclass (page_number, image_b64, image_path). Used by legacy pipeline for image-based OCR.
📊 Models
models.py
Data models: ContentBlock, PageResult, DocumentMetadata, DocumentResult. Structured output matching OCR JSON schema. to_dict() serialization. Feature detection.
💾 Output Handler
output_handler.py
Save JSON results. Per-page JSON files. Merged document JSON. Filename sanitization. Directory auto-creation. UTF-8 encoding for Bangla.

💬 OCR Prompts

TURBO_PROMPT (Compact — turbo_pipeline.py)

Rule Details
Output Format JSON with: page, blocks[], tables[], full_text, notes[]
Block Types header, paragraph, list, table, image, signature
Languages bn (Bangla), en (English), mixed
Confidence high, med, low
Bengali Numerals Verify: ০১২৩৪৫৬৭৮৯ — check ৩ vs ৫, ৮ vs ৪
Images Describe what you see, never "[IMAGE]"
Handwritten Set handwritten=true, describe signatures
Tables Extract all cells as 2D array

MASTER_PROMPT (Full — ocr_engines.py)

Section Details
Bengali Numerals (HIGHEST PRIORITY) Visual characteristics of each digit ০-৯. Most confused pairs: ৩↔৫, ৮↔৪, ৬↔৯, ২↔৩. 6-step extraction protocol.
Text Accuracy Preserve mattra, hasanta, nukta, conjuncts. No transliteration/translation.
Handwriting Signatures → describe or "[Signature of ...]". Complete partial words contextually. Mark [ILLEGIBLE] when needed.
Layout Detection Tables, forms, columns, boxes, lists, headers/footers, page numbers, stamps.
Image Description Logos, seals, photos → detailed description (never "[IMAGE]").
Output JSON page_number, content_blocks[], tables[], forms[], full_text_reading_order, extraction_notes[]
